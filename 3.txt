Assignment 3: Docker Volumes & Dockerfiles
Q1. Docker Volume Persistence   

Create volume: docker volume create data_vol   

Run container: docker run -d --name test1 -v data_vol:/data ubuntu sleep 1000   

Create file and remove container: docker exec test1 bash -c "echo 'hello' > /data/file.txt" followed by docker rm -f test1.  

Start new container: docker run -it --rm -v data_vol:/data ubuntu cat /data/file.txt   

The file will still exist because the volume persists independently of the container lifecycle.  

Q2. Dockerfile - Python Application   

Create app.py: print("Hello from Docker")   

Create Dockerfile:  

Dockerfile
FROM python:3.9-slim
COPY app.py /app.py
CMD ["python", "/app.py"]

Build and run: docker build -t my-python-app . followed by docker run my-python-app.

Q3. Dockerfile - Java Application   

Create Main.java: public class Main { public static void main(String[] args) { System.out.println("Hello Java"); } }   

Create Dockerfile:  

Dockerfile
FROM openjdk:11
COPY Main.java /
RUN javac Main.java
CMD ["java", "Main"]
Build and run: docker build -t my-java-app . followed by docker run my-java-app.  

Q4. Docker Volume with MySQL   

Run MySQL: docker volume create db_vol && docker run -d --name mysql_db -e MYSQL_ROOT_PASSWORD=root -v db_vol:/var/lib/mysql mysql:latest   

Insert data: docker exec -it mysql_db mysql -uroot -proot -e "CREATE DATABASE testdb; USE testdb; CREATE TABLE users(id INT); INSERT INTO users VALUES(1);"   

Restart and verify: docker restart mysql_db and then docker exec -it mysql_db mysql -uroot -proot -e "USE testdb; SELECT * FROM users;" to verify data persists.  

Q5. Dockerfile - Nginx Web Server   

Create index.html: <h1>My Custom Nginx Page</h1>   

Create Dockerfile:  

Dockerfile
FROM nginx:latest
COPY index.html /usr/share/nginx/html/index.html
Build and run: docker build -t custom-nginx . followed by docker run -d -p 8080:80 custom-nginx.  

Verify by navigating to http://localhost:8080 in a browser.  

Q6. Bind Mount vs Volume   

Create directory/file: mkdir host_data && echo "data" > host_data/file.txt   

Run with bind mount: docker run -it --rm -v $(pwd)/host_data:/container_data ubuntu cat /container_data/file.txt   

Difference: A bind mount links a specific, user-defined path on the host machine to the container, whereas a Docker volume is fully managed by Docker and stored in a specialized directory, making volumes safer and easier to migrate.  

Q7. Dockerfile - Shell Script Execution   

Create script.sh:  

Bash
#!/bin/sh
echo "Current System Date:"
date
Create Dockerfile:  

Dockerfile
FROM alpine
COPY script.sh /script.sh
RUN chmod +x /script.sh
CMD ["/script.sh"]
Build and run: docker build -t shell-app . followed by docker run shell-app.  

Q8. Shared Volume Between Containers   

Create volume and run containers: docker volume create shared_vol, then docker run -d --name c1 -v shared_vol:/shared ubuntu sleep 1000 and docker run -d --name c2 -v shared_vol:/shared ubuntu sleep 1000.  

Write data: docker exec c1 bash -c "echo 'Shared Text' > /shared/data.txt"   

Read data: docker exec c2 cat /shared/data.txt
